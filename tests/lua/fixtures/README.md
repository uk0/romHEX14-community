# Lua test fixtures

Test fixtures live here at runtime but are **not committed** — they're
either customer-supplied ECU images or regenerated by tests.

## Required fixture

`fixture.bin` — any 2 MB ECU image. The Lua test runner
(`tests/lua/run_tests.ps1`) sources it via:

1. `$env:ROMHEX14_FIXTURE_BIN` — if set and the file exists, copy to
   `fixture.bin`.
2. Otherwise reuse an existing `fixture.bin` already in this directory.
3. Otherwise the runner aborts with an error.

The current test suite assumes a Bosch EDC17C46 image, but any 2 MB
binary will exercise the byte-level APIs. A few tests reference a
specific MD5 (`test_stress_getproperty_md5.lua`); skip or update that
test if you use a different binary.

## Generated artifacts

Tests that exercise `projectExport` / `MapListExporter` write under
`tmp_*` names here:

* `tmp_export.bin`, `tmp_proj_export.bin`, `tmp_roundtrip.bin` ...
* `tmp_maps.csv`, `tmp_maps.json`, `tmp_proj_maps.json` ...
* `tmp_http_response.bin` — HTTP test capture
* `tmp_create_test/` — directory-mode test

All `tmp_*` patterns are gitignored. The runner overwrites them on
every invocation.
