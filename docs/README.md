# romHEX 14 тАФ User Manual (local build pipeline)

This directory contains the source and tooling for the romHEX 14 user manual.
**Everything builds locally on your machine.** The built output (HTML site +
PDFs) is then pushed to the public repo
[ctabuyo/romHEX14-ECU-Tuning](https://github.com/ctabuyo/romHEX14-ECU-Tuning).

The Markdown source is **not** committed to the public repo тАФ only the rendered
HTML and PDFs are.

## First-time setup

```bash
cd docs
make install            # creates .venv, installs Python deps
export ANTHROPIC_API_KEY=sk-ant-...
```

## The pipeline

```bash
make gen          # regenerate glossary, menu reference, changelog
make screenshots  # capture all screenshots in EN + zh-CN  (see "Screenshots" below)
make translate    # Claude ENтЖТzh-CN, only translates changed files
make site         # build HTML site
make pdf          # build per-language PDFs
make docs         # = gen + translate + site + pdf  (the full pipeline)
make serve        # live preview at http://127.0.0.1:8000
make publish      # copy site/ + pdf/ into the public repo
make clean        # remove build artefacts
```

A typical iteration loop while writing a chapter:

```bash
# edit src/en/10-map-editor.md
make serve        # live preview, no translation needed
# happy with the EN content?
make translate    # generates src/zh/10-map-editor.md
# review/fix the zh version, then:
make docs         # final build with PDFs
make publish      # push to public repo
```

## Generators (`make gen`)

`tooling/gen_docs.py` produces three files in **both** languages:

| File | Source |
|---|---|
| `appendix-a-menus.md` | regex over `src/*.cpp` for `QAction` + `setShortcut` |
| `appendix-c-glossary.md` | parsed from `translations/rx14_en.ts` + `rx14_zh_CN.ts` |
| `appendix-d-changelog.md` | `git log` of the repo root |

These files are listed in `.gitignore` and recreated on every `make gen` run.

## Translation (`make translate`)

`tooling/translate.py` uses the Claude API to translate every Markdown file in
`src/en/` to `src/zh/`, with these properties:

- **Diff-based**: a SHA-256 hash of each EN file is cached in
  `.translate-cache/hashes.json`. Files whose hash hasn't changed are skipped.
- **Glossary-aware**: the auto-generated `appendix-c-glossary.md` is injected
  into the Claude system prompt so UI terminology stays consistent with the
  app's actual zh-CN strings.
- **Skips auto-generated appendices** (A, C, D) тАФ those are already produced
  bilingually by `gen_docs.py`.
- **Resumable**: cache is saved after each successful file, so you can Ctrl-C
  and resume.
- **Force re-translate**: `python tooling/translate.py --force тАж`

## Screenshots (`make screenshots`)

This is the only piece that is **not yet implemented** тАФ `run_screenshots.py`
is currently a stub. The manifest in `tooling/screenshots.yaml` defines 40+
screenshots covering every major dialog and workflow, using the demo files in
`C:/Users/Cris/Desktop/чЙ╣ш░Г` as fixtures.

There are two implementation strategies (documented in detail at the top of
`run_screenshots.py`):

1. **Strategy A тАФ in-app screenshot mode** (recommended): add a
   `--screenshot-mode <manifest.yaml>` flag to `rx14.exe`. The app reads the
   manifest, runs each scenario programmatically and calls `QWidget::grab()`.
   Permanent, deterministic, headless-capable.
2. **Strategy B тАФ pywinauto/pyautogui**: drive the running app from outside.
   Faster to prototype, fragile in the long run.

**Recommended path:** start with Strategy B for the 5тАУ10 most critical
screenshots so the manual has visuals immediately, then migrate to Strategy A
for permanent infrastructure.

## What's committed vs what's gitignored

**Committed to the private repo** (`rx14-qt`):
- `mkdocs.yml`, `requirements.txt`, `Makefile`, `README.md`, `.gitignore`
- `src/en/*.md` and `src/zh/*.md` (hand-written chapters)
- `tooling/*.py` and `tooling/screenshots.yaml`

**Gitignored**:
- `.venv/`, `__pycache__/`, `*.pyc`
- `site/`, `pdf/` (build outputs)
- `.translate-cache/` (translation hashes)
- `src/*/assets/screenshots/` (large binary PNGs тАФ re-generate with `make screenshots`)
- `src/*/appendix-{a,c,d}-*.md` (auto-generated)

**Pushed to the public repo** (`romHEX14-ECU-Tuning`) by `make publish`:
- `site/*` тЖТ `docs/`
- `pdf/*.pdf` тЖТ `docs/pdf/`

The public repo never sees the Markdown source or the tooling тАФ only the
finished HTML + PDFs.
